# Security Policy

This repository contains public, outcome-level documentation only. Do not submit private keys, seed phrases, service-role keys, RPC keys, deployment scripts, operator procedures, internal logs, or unreleased implementation details in issues, pull requests, comments, or attachments.

## Public Reporting Scope

Public security discussion should stay at the level of:

- documentation errors,
- broken links,
- misleading production-readiness language,
- public address corrections,
- general protocol risk categories.

## Private Reporting

Potential vulnerabilities, exploit paths, private infrastructure details, or implementation-specific findings should be sent through the maintainer channel approved by the Orina Protocol owner. Do not post exploit instructions or private operational details publicly.
