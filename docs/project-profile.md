## Overview

Orina Protocol is a settlement layer for bilateral commerce. Its Atomic Transaction Protocol (ATP) gives independent parties a shared path for escrowed, time-bounded, dispute-aware transactions that resolve into one final settlement outcome.

The protocol is designed to support marketplaces, wallets, dashboards, partner applications, and automation-ready commerce interfaces that need a common settlement layer instead of rebuilding transaction finality for every application.

## What Orina Standardizes

ATP standardizes the public transaction outcome layer around:

- transaction initiation,
- escrowed payment handling,
- seller and buyer settlement states,
- timeout handling,
- dispute boundary,
- release or refund outcome,
- final settlement record.

## What Remains Application-Specific

Applications built around Orina can keep their own:

- product discovery and search,
- messaging and negotiation experience,
- user interface,
- asset category,
- seller onboarding,
- legal and compliance process,
- logistics and fulfillment workflow,
- payment rail selection where supported.

## Public Architecture Summary

At the public documentation level, Orina can be described as:

1. A transaction lifecycle model for bilateral commerce.
2. An escrow-oriented settlement authority for supported transactions.
3. A final outcome record that applications can reference.
4. A testnet-operated protocol surface being prepared for broader security review and production governance.

This repository does not publish private implementation mechanics, source-level contract internals, private deployment maps, or wallet automation design material.
