# Public Disclosure Boundary

---
date: 2026-07-02
status: Public repository boundary
---

This repository is the public documentation boundary for Orina Protocol. It is designed to explain outcomes, public posture, and selected references without releasing private implementation material.

## Included

| Category | Public treatment |
| --- | --- |
| Protocol positioning | High-level description of ATP as a settlement outcome layer |
| Public outcomes | Lifecycle, escrow, timeout, dispute, release, refund, and final record concepts |
| Selected addresses | Three primary ATP testnet contracts only |
| Tokenomics | ORI public allocation, unlock, and fee-alignment reference |
| Readiness | Public distinction between testnet evidence and production approval |
| Formal and audit posture | Summary-level assurance status without model files, bytecode, traces, or internal findings detail |
| Links | Official website, runtime app, whitepaper, social channels, and this repository |

## Excluded

| Category | Reason |
| --- | --- |
| Source-level contract mechanics | Protected implementation detail |
| Full deployment maps | Not approved for public release |
| Auxiliary contract addresses | Not part of the approved public address sheet |
| Deployment scripts and runbooks | Operational security boundary |
| Private wallet automation design | Protected product and IP boundary |
| Formal artifacts, bytecode dumps, traces, and internal logs | Verification and operations boundary |
| Secrets, keys, service credentials, and RPC configuration | Security boundary |

## Public Review Rule

Public reviewers should evaluate this repository as an outcome, readiness, and ecosystem reference. Requests for deeper implementation review should move through a private maintainer-approved process.

## Language Rule

Public descriptions should prefer:

- "settlement outcome layer"
- "escrowed transaction lifecycle"
- "testnet-operated"
- "production readiness pending"
- "formal assurance summary"
- "selected public ATP addresses"
- "automation-ready commerce interfaces"

Public descriptions should avoid claims that imply final production approval, external audit completion, guaranteed real-world delivery, investment returns, or disclosure of private implementation mechanics.
